변수의 명명법

  • 변수의 이름은 영문자(대소문자)와 숫자, '_','$'만 사용
  • 변수의 시작은 숫자를 제외한 나머지로 시작
  • 변수의 이름에 2 단어 이상이 합쳐질 경우 Camel Expression 적용
  • 변수명은 다른 사람이 보더라도 변수의 의미를 이해할 수 있는 이름을 붙임
  • 예약어(var, let, null, undefined, number, for, if ...)는 제외

변수의 할당(선언+초기화)

  • num = 10 // 암묵적 전역공간에 변수가 생겨서 진행시 충돌 가능성(실무 비추)
  • var num = 10 // 의도하여 선언하는 방식
  • let num = 10
  • const num = 10 // 선언과 초기화의 분리 가능
    • 제어자(Modifier) 재선언 재할당 Scope(범위)
    var ok ok 함수 scope
    let no ok 블록 scope
    const no no 블록 scope

변수의 선언과 호이스팅

  • hoisting
    • 자바스크립트는 실행 전에 변수 선언을 먼저 처리하고 마치 코드 상단에 선언한것처럼 사용한다. 그래서 변수 선언만 상단으로 가져가고 초기화는 해당 라인에서 진행하는 형태
  • 특징 : 선언은 상단으로, 초기값은 undefined로, 에러는 안나게
  • var은 호이스팅을 하게 되면 undefined로 처리되지만, let 과 const는 error 발생
    • 그렇다고, let 과 const가 호이스팅이 되지 않는 것은 아니다. 단지 접근만 안됨.
